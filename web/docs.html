Prerequisites

  • macOS or Linux
  • Bun 1.0+ to run the chan launcher
  • One configured model provider (API Key or local Ollama)
  • Go 1.26+ (Required only for Source Build)

Installation

Downloads prebuilt binaries automatically. Bun 1.0+ must already be installed.

$curl -fsSL https://raw.githubusercontent.com/channyeintun/chan/main/chan/install.sh | sh

Installs chan and chan-engine to /usr/local/bin or ~/.local/bin.

Manual Installation

A: Global Install

sudo install -m 755 chan /usr/local/bin/chan
sudo install -m 755 chan-engine /usr/local/bin/chan-engine

B: Local User (No Sudo)

mkdir -p "$HOME/.local/bin"
install -m 755 chan "$HOME/.local/bin/chan"
install -m 755 chan-engine "$HOME/.local/bin/chan-engine"

GitHub Copilot

Run /connect within the TUI. For Enterprise domains:

/connect github-copilot your-company.example
  • • Prints device code in transcript
  • • Opens verification URL automatically
  • • Authorizes v5.4 models

First-Class Outputs

Artifacts in Chan are durable, structured outputs that stay reviewable in the main TUI instead of disappearing into the transcript.

  • Implementation Plans: Proposals before execution.
  • Task Lists: Real-time progress trackers.
  • Walkthroughs: Post-completion summaries.
  • Durable Returns: Large fetch/diff results.

CLI Options

# Use a specific model
$ chan --model openai/gpt-4o
# Direct execution (skip planning)
$ chan --mode fast

Diagnostics

Chan now supports both startup debug capture and live session capture. The canonical log format is structured JSONL so you can tail it directly or pipe it into external monitoring tools.

From Launch

CHAN_DEBUG=1 chan

Inside The TUI

/debug
/debug status
/debug path
/debug off

Manual Monitor

chan debug-view --file ~/.config/chan/sessions/<session-id>/debug.log
tail -F ~/.config/chan/sessions/<session-id>/debug.log | jq .
  • • On macOS, /debug opens a new Terminal window with the live monitor attached to the current session log.
  • • Each line in debug.log is a structured event envelope with fields like schema_version, ts, session_id, component, event, level, metrics, data, and error.
  • • Sensitive raw payloads are redacted and truncated before they are written to the log.

Architecture

Chan uses a split-process architecture. A Bun-powered TUI communicates with a Go engine over NDJSON. Every turn, the engine selects live context through a session-scoped retrieval graph — no cached code summaries.

Building from Source

bun run setup       # Build TS & compile Go
bun run start       # Run dev TUI
make release-local  # Package per-platform bin
