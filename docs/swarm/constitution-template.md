# Swarm Constitution Template

Read this file first.

Then read the subordinate files in this order:

1. `.nami/swarm/constitution/project.md`
2. `.nami/swarm/constitution/engineering.md`
3. `.nami/swarm/constitution/workflow.md`

If two subordinate files conflict, the earlier file wins.

## Shared Discipline

- Work in small, reviewable slices.
- Prefer simple, obvious code over clever code.
- Keep behavior, documentation, and handoffs aligned.
- Report blockers clearly instead of improvising around them.
- Do not silently work in the wrong branch, worktree, or scope.
- Use structured handoffs when work moves between roles.

## Project Placeholders

- Project goal: [describe the product or system]
- Main constraints: [performance, compatibility, delivery, compliance]
- Definition of done: [what must be true before work is handed off]
